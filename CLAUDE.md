## Quick Start

```bash
pip install -e ".[dev,web]"
swing --help                         # CLI: trade, journal, pipeline, web, finviz, etc.
swing web                            # FastAPI + HTMX dashboard on 127.0.0.1:8080
python -m pytest -m "not slow" -q    # fast suite (~7038 tests as of 2026-06-02, ~3min with xdist)
python -m pytest -m slow             # pipeline/yfinance e2e (minutes, needs network)
ruff check swing/
```

**Windows PATH for `swing` CLI:** `pip install -e` places the `swing.exe` entry point in `%APPDATA%\Python\Python314\Scripts\` (user install, not `C:\Python314\Scripts`). Add that dir to the user PATH permanently via System Properties, or per-session: `$env:PATH = "$env:APPDATA\Python\Python314\Scripts;$env:PATH"`.

**Finviz inbox:** `data/finviz-inbox/` (configured in `swing.config.toml`). Pipeline now fetches the screen via Finviz Elite API automatically (`_step_finviz_fetch`); manual CSV drop remains a fallback override (file-collision skip in the pipeline step). Save manual exports as `finvizDDMmmYYYY.csv` (1-2 digit day, 3-letter month, 4-digit year — e.g. `finviz5May2026.csv`). API-emitted CSVs use the same filename pattern, anchored on `action_session_for_run(datetime.now())` per [`swing/pipeline/runner.py:_step_finviz_fetch`](swing/pipeline/runner.py). Validator requires 13 columns: `No., Ticker, Sector, Industry, Country, Price, Change, Average Volume, Relative Volume, Average True Range, 52-Week High, 52-Week Low, Market Cap`. Missing columns → CSV moved to `data/finviz-inbox/rejected/` with a sidecar `.rejected-reasons.json`. Pipeline outputs go to `exports/<action_session_date>/` (briefing.md + briefing.html + charts/).

**Pipeline lease-acquisition wait** (`web.pipeline_lease_wait_seconds`, default 5s): how long POST `/pipeline/run` waits for the spawned subprocess to insert its `pipeline_runs` row. Python 3.14 cold-start + heavy imports on Windows regularly cross 2s; bump if you see false "did not acquire lease" errors on a slow box.

> **Code / runtime / test failure-prevention, compressed to trigger + fix** (restructured 2026-05-28). Full forensic provenance (dates, commit SHAs, Codex rounds, findings docs) for every entry → [`docs/CLAUDE.md-archive.md`](docs/CLAUDE.md-archive.md) §"Appended 2026-05-28". The **process / review + brief-authoring disciplines** (the "Expansion #N" catalog + applied-research methodology) were relocated to [`docs/orchestrator-context.md`](docs/orchestrator-context.md) §"Pre-Codex review + brief-authoring disciplines". `#N` labels below are the cumulative-gotcha numbers referenced elsewhere in briefs.

### yfinance / market-data

- **yfinance rate-limits.** Use `threads=False` on `yf.download()` ONLY. `yf.Ticker(t).history()` does NOT accept `threads=` (raises `TypeError` on yfinance >= 1.2). Concurrency for `Ticker.history()` is bounded by the app executor; `PriceCache`/`OhlcvCache` sliding-window breakers are the real backpressure.
- **yfinance `history(interval="1d")` includes the in-progress (partial) bar during market hours.** Strip it before computing SMAs / "previous close": `if df.index[-1].date() >= session: df = df.iloc[:-1]` where `session = action_session_for_run(datetime.now())`. Use the exchange-session helper, NOT `date.today()` (HST lags ET 5h).
- **yfinance `group_by='column'` returns a MultiIndex column** (`Price × Ticker`) even for single-ticker calls — `df["Close"]` is a DataFrame, not Series. Squeeze: `close = df["Close"]; if getattr(close,'ndim',1)==2: close = close.iloc[:,0]`.
- **`exchange_calendars.is_open_at_time` requires `pd.Timestamp`**, not `datetime.datetime` (raises `TypeError`). Wrap: `pd.Timestamp(datetime.now(timezone.utc))`. Log dependency-call exceptions instead of silently returning a degraded default.
- **Schwab `price_history` defaults to `(periodType=day, period=10, frequencyType=minute, frequency=1)`** — 10 days of 1-MINUTE bars, not daily. Any consumer wanting DAILY bars MUST pass `(year|month, N, daily, 1)` explicitly. Discriminating test asserts the kwargs via mock/signature.
- **OHLCV fetch scope = open-trade tickers ONLY.** `build_dashboard` computes `ohlcv_tickers = sorted({t.ticker for t in open_trades})` — do NOT union watchlist/active_tickers (burns quota, trips the breaker). Skip the fetch entirely with zero open trades.
- **External-API empty result must be treated as transient in write-through caches/archives** (F6). yfinance/broker/weather APIs return empty for rate-limit/network/weekend reasons — NEVER overwrite cached content with an empty response: retain existing, leave stale metadata so the next call retries, never blank. Applies to `swing/data/ohlcv_archive.py` + any future write-through cache. When the cache helper accepts a **dataclass**, enforce non-empty at the `__post_init__` construction barrier (rejecting in the helper is too late — the row is already built).
- **Shared-infrastructure cache hooks must return the FULL archive; consumers slice.** A fallback that pre-truncates to one consumer's minimum window silently caps every other consumer (e.g. chart needs ~145 business days vs SMA50's 60). Return full history; let each consumer slice.

### Web / HTMX / templates / forms

- **HTMX 4xx fragments need an explicit config override.** HTMX 2.x default (`{code:"[45]..",swap:false}`) fires the error event but doesn't swap. `base.html.j2` has an `htmx.config.responseHandling` override enabling 4xx swapping — preserve it. TestClient won't catch a regression (it asserts body, not DOM).
- **Starlette middleware is LIFO.** `add_middleware` order is the REVERSE of request execution — later calls become more-outer. Security-critical middleware (OriginGuard strict-mode, request-id) must be added LAST so it wraps everything.
- **`base.html.j2` is shared — a new `vm.foo` field requires adding it (with a safe default) to EVERY base-layout VM** (`DashboardVM`, `PipelineVM`, `JournalVM`, `WatchlistVM`, `PageErrorVM`) or Jinja 500s unrelated routes with `UndefinedError`.
- **Cache + executor race: workers must not write shared state when the request thread cancels on deadline miss.** `Future.cancel()` is a no-op on a running worker. Worker returns a pure value; the request thread writes to `_store` ONLY for futures done in-deadline (`wait(timeout=deadline)`, iterate `done` not `pending`).
- **HTMX OOB-swap partials that hand-duplicate full-page markup drift silently.** OOB partials and full-page render must go through the SAME `{% include %}` target, never hand-duplicated fragments.
- **HTMX response leading with `<tr>` triggers `makeFragment` synthetic-table-wrap that DROPS `<table>` content inside OOB `<section>` chunks** (HTML5 nested-table parse rules; only in a real browser, not TestClient). Keep any new HTMX endpoint's primary content table-row-free at fragment root (use `<div>`/`<section>`); deliver new rows via OOB swap into the destination tbody.
- **HTMX `hx-target` INHERITS from ancestors — a lazy child element MUST set `hx-target="this"` when an ancestor already sets `hx-target`.** A `<td>` with `hx-get`+`hx-trigger="revealed"`+`hx-swap="innerHTML"` but NO explicit `hx-target`, inside a `<tr hx-target="closest tr">` (click-to-expand), inherits the row's target → the revealed lazy-load swaps its response into the WHOLE ROW's innerHTML, wiping every other cell (and breaking expand). Set `hx-target="this"` on the child so it swaps into itself. Browser-only (the initial render looks correct; only the live trigger fires it; caught at the Phase-14 close-out P14.N1 operator gate). A TestClient regression asserts the rendered child carries `hx-target="this"` (the exact attribute order, so it won't false-match the ancestor's target).
- **HTMX form endpoints have two browser-only failure surfaces TestClient cannot detect:** (1) embedded forms inside HTMX fragments need `hx-headers='{"HX-Request": "true"}'` or OriginGuard strict-mode 403s the submit; (2) success path must be `204` + `HX-Redirect: <url>`, NOT `303`→swap-target (htmx.js swallows 303). Operator-witnessed browser verification is BINDING for HTMX work.
- **HX-Redirect target route must be verified to exist** — TestClient verifies the header but doesn't follow. Assert the target is in `app.routes` OR follow the redirect with a second call asserting 200.
- **Matplotlib mathtext fires on `$` `^` `_` and unbalanced `\`** — silently italicizes text, eats the metacharacters, collapses whitespace in the RENDERED PNG. String-equality tests check the format string, not the render. Fix: omit the metacharacter from any `title=`/`xlabel=`/`ylabel=`/annotation entirely (escaping `\$` still triggers math mode). Manual visual verification of rendered text is non-optional.
- **Weather lookup in read-only UIs must NOT query by `action_session`.** Weather is keyed by `data_asof_date` (last completed session); `action_session` is forward-looking → silent `None` on weekends/evenings. Use `swing.data.repos.weather.get_latest(conn, ticker=...)` (most recent by `run_ts`).
- **Session-anchor read/write mismatch** (forward-looking `action_session_for_run` vs backward-looking `last_completed_session`) silently invisibles UI display. Recurring family (weather, daily-mgmt badge, in-progress-bar). Read the WRITER's anchor function before locking a read predicate; add a write-then-read round-trip test. Inequality directionality: use `>=` for forward-looking anchors, strict `>` for backward-looking (the boundary bar is inclusive backward).
- **V1 single-operator forms with hidden audit fields (timestamps, session anchors, precision tiers): SERVER-STAMP, don't trust hidden inputs.** And "server-stamped" means the POST handler RE-COMPUTES from canonical state at POST time — NOT that a GET-rendered value is trusted on resubmit. Remove the hidden input; expose computed values as display-only `<span>`.
- **Form-render hidden anchors that drive POST-time validation MUST round-trip through the soft-warn confirm `form_values` dict** or a `force=true` resubmit drops the anchor → validation silently bypassed.
- **POST-time recompute of "latest-of-something" creates a GET/POST TOCTOU window.** If the operator saw the value at form-render time, emit it as a hidden anchor and validate the POST against that exact anchor's row — don't recompute "latest" at POST time.
- **Hidden-anchor 4-tier rejection ladder** for web forms with audit anchors: (a) malformed JSON → 400 + clear; (b) non-dict → 400 + clear; (c) missing required keys → 400 + clear; (d) invalid value shapes → 400 + clear. Plus a `claimed_auto_fill` consistency gate. On rejection the recovery form MUST clear the bad anchor (pass `submitted_*=None`), else the operator is trapped replaying it.
- **`selected_X_audit_id` is an AUDIT TRAIL, not a DEDUPE KEY.** Dedupe off the field reflecting the row's value-source (what was persisted), not what the operator picked.
- **Price-precision parity between template render and POST float comparison.** If the template renders `%.2f` and the POST compares with a tight epsilon, execution-grain prices falsely flip to "operator edited". Compare with `round(anchor,N) != round(submitted,N)` where N matches display precision.
- **`extended.pop(key, None)` for envelope keys that may not apply** — don't leave a stale form-render default; pop it so downstream dedupe falls through to the fallback match.
- **Audit-envelope empty state must be uniform `None` (not `"[]"`, not `""`) across BOTH the VM emit path and the persist write path** — `"[]"` is truthy and defeats the `... or None` SQL-nullability defense.

### SQLite / transactions / migrations / schema

- **`os.replace` requires same filesystem.** On Windows with Drive-synced dest + `$TMP` on another volume it raises `OSError 18`. Create temp files in the DESTINATION dir (`NamedTemporaryFile(dir=<dest_dir>)`). Never `shutil.move` expecting overwrite on Windows.
- **Python `... or ""` collides with SQL CHECK-constraint nullability** when the enum excludes empty string. Use `... or None` for nullable text columns with CHECK constraints; add a test submitting an empty field asserting NULL persisted.
- **SQLite `INSERT OR REPLACE` / bare `REPLACE` is `DELETE old + INSERT new`** — cascade-wipes child rows via `ON DELETE CASCADE`, issues a NEW PK, rewrites history. For audit/uniqueness tables use SELECT-then-UPDATE-or-INSERT.
- **Service-layer `with conn:` opens its own transaction** — do NOT call it from inside an outer single-transaction (inner context-manager commits the connection's single transaction). When composing multi-step atomic side-effects, call the REPO-level functions directly; preserve the repo-vs-service asymmetry.
- **(#9) `sqlite3.executescript()` issues an implicit COMMIT and runs statements in autocommit** — `conn.rollback()` can't undo intermediate statements. Migration runner MUST do explicit `BEGIN`+`executescript`+`COMMIT` with try/except `rollback()`+re-raise (`swing/data/db.py:_apply_migration`, foreign_keys=OFF at runner level). Test rollback through the real runner path, not bare `executescript`.
- **`in_transaction` "auto-detect outer transaction" guards re-introduce the race the explicit lock closed.** Single-transaction service contract: caller MUST NOT hold an open tx; function ALWAYS owns `BEGIN IMMEDIATE`/COMMIT/ROLLBACK; REJECT (don't auto-detect) caller-held transactions. Environment short-circuits (sandbox/dry-run) live in the INNER (caller-tx) function so the outer tx-discipline (reject-caller-held / BEGIN IMMEDIATE) stays uniform.
- **Reconciliation flow pivot uses SAVEPOINT-per-discrepancy inside the outer run transaction** (`SAVEPOINT correction_sp_<id>` / RELEASE on success / ROLLBACK TO+RELEASE on exception) — without it a partial mutation commits on the outer COMMIT.
- **SELECT-first idempotency MUST precede payload validation** — a terminal-state row must return its existing audit-row id even with a stale/None payload. Step order: sandbox short-circuit → SELECT + None-check → terminal-state idempotent return → payload validation → atomic write.
- **Counter staleness after inline state mutation** — a flow that emits rows incrementing a counter AND later mutates those rows' states must decrement per-mutation OR recompute via `SELECT COUNT(*)` post-loop.
- **Queries `ORDER BY started_ts DESC` on `pipeline_runs` mask prior completes mid-run** (a NULL-`finished_ts` row wins). Two-read pattern: most-recent COMPLETED row's `finished_ts` for "last good data"; most-recent-started row's `state` for "what's happening now".
- **(#11) Schema-CHECK + Python-constant + dataclass-validator MUST land in ONE task** for atomic consistency — and this EXTENDS to semantic contracts layered atop the CHECK (cache-key shapes, partial-index existence semantics): mirror EACH in `__post_init__`, not just the schema CHECK. When widening a CHECK enum, grep ALL `swing/` for hardcoded copies of the old tuple/enum (multiple Python mirrors: constants, `__post_init__`, repo guards, per-surface `_call_endpoint` guards) AND audit each as manual-input-allowlist (tight) vs service-only (full-widened). Read-path `_row_to_*` mappers MUST be widened in the same task as the write-path. Schema-version-aware INSERT: detect version via `PRAGMA table_info` and branch column-lists OR migrate all pre-version fixtures.
- **Migration backup-gate uses STRICT equality `pre_version == (target - 1)`, NOT `<=`** (multi-version jumps must be separate two-step migrations). Copy the Phase 9 `pre_version == 16 AND target >= 17` clause shape verbatim.
- **sqlite3 cannot bind a list to a single `:name` placeholder** — build dynamic `?` expansion for IN clauses (`",".join("?"*len(values))`); short-circuit empty input (empty `IN ()` is invalid SQL) and return a dataclass with all required sentinel fields.
- **`date.fromisoformat()` at the TEXT-column → Python-`date` boundary.** When a SQL TEXT column feeds a helper expecting `date`, convert at the callsite; add a malformed-input test asserting a typed error, not a deep `TypeError`.
- **`reconciliation_corrections` is APPEND-ONLY** — tier-3 chains INSERT a new row + UPDATE the prior row's `superseded_by_correction_id`; never DELETE/UPDATE-in-place. Chain head has `superseded_by_correction_id IS NULL`.
- **`PriceCache._last_close` only sees tickers in today's `candidates` table.** Open-trade tickers that rotate out of finviz get stale closes; `_step_evaluate` unions `list_open_trades` and writes them `bucket='excluded'` to keep closes fresh. New "priced" surfaces must verify their tickers appear in `candidates`.
