```bash
pip install -e ".[dev,web]"
swing --help                         # CLI: trade, journal, pipeline, web, finviz, etc.
swing web                            # FastAPI + HTMX dashboard on 127.0.0.1:8080
python -m pytest -m "not slow" -q    # fast suite (~7038 tests as of 2026-06-02, ~3min with xdist)

```

**Windows PATH for `swing` CLI:** `pip install -e` places the `swing.exe` entry point in `%APPDATA%\Python\Python314\Scripts\` (user install, not `C:\Python314\Scripts`). Add that dir to the user PATH permanently via System Properties, or per-session: `$env:PATH = "$env:APPDATA\Python\Python314\Scripts;$env:PATH"`.

**Finviz inbox:** `data/finviz-inbox/` (configured in `swing.config.toml`). Pipeline now fetches the screen via Finviz Elite API automatically (`_step_finviz_fetch`); manual CSV drop remains a fallback override (file-collision skip in the pipeline step). Save manual exports as `finvizDDMmmYYYY.csv` (1-2 digit day, 3-letter month, 4-digit year — e.g. `finviz5May2026.csv`). API-emitted CSVs use the same filename pattern, anchored on `action_session_for_run(datetime.now())` per [`swing/pipeline/runner.py:_step_finviz_fetch`](swing/pipeline/runner.py). Validator requires 13 columns: `No., Ticker, Sector, Industry, Country, Price, Change, Average Volume, Relative Volume, Average True Range, 52-Week High, 52-Week Low, Market Cap`. Missing columns → CSV moved to `data/finviz-inbox/rejected/` with a sidecar `.rejected-reasons.json`. Pipeline outputs go to `exports/<action_session_date>/` (briefing.md + briefing.html + charts/).

**Pipeline lease-acquisition wait** (`web.pipeline_lease_wait_seconds`, default 5s): how long POST `/pipeline/run` waits for the spawned subprocess to insert its `pipeline_runs` row. Python 3.14 cold-start + heavy imports on Windows regularly cross 2s; bump if you see false "did not acquire lease" errors on a slow box.

## Gotchas

> **Code / runtime / test failure-prevention, compressed to trigger + fix** (restructured 2026-05-28). Full forensic provenance (dates, commit SHAs, Codex rounds, findings docs) for every entry → [`docs/CLAUDE.md-archive.md`](docs/CLAUDE.md-archive.md) §"Appended 2026-05-28". The **process / review + brief-authoring disciplines** (the "Expansion #N" catalog + applied-research methodology) were relocated to [`docs/orchestrator-context.md`](docs/orchestrator-context.md) §"Pre-Codex review + brief-authoring disciplines". `#N` labels below are the cumulative-gotcha numbers referenced elsewhere in briefs.

### yfinance / market-data

- **yfinance rate-limits.** Use `threads=False` on `yf.download()` ONLY. `yf.Ticker(t).history()` does NOT accept `threads=` (raises `TypeError` on yfinance >= 1.2). Concurrency for `Ticker.history()` is bounded by the app executor; `PriceCache`/`OhlcvCache` sliding-window breakers are the real backpressure. **ONE authorized exception (operator, 2026-06-10):** the Arc-6 warm's multi-ticker BATCH chunk fetch (`_fetch_chunk`, ohlcv_archive.py) runs `threads=True` — the spec §8 stretch lever; safe because the warm is a pure accelerator with a proven per-ticker serial fallback (worst case = a slower night). The SINGLE-ticker path (`_yf_download_window`) stays `threads=False`.
- **yfinance `history(interval="1d")` includes the in-progress (partial) bar during market hours.** Strip it before computing SMAs / "previous close": `if df.index[-1].date() >= session: df = df.iloc[:-1]` where `session = action_session_for_run(datetime.now())`. Use the exchange-session helper, NOT `date.today()` (HST lags ET 5h).
- **yfinance `group_by='column'` returns a MultiIndex column** (`Price × Ticker`) even for single-ticker calls — `df["Close"]` is a DataFrame, not Series. Squeeze: `close = df["Close"]; if getattr(close,'ndim',1)==2: close = close.iloc[:,0]`.
- **`exchange_calendars.is_open_at_time` requires `pd.Timestamp`**, not `datetime.datetime` (raises `TypeError`). Wrap: `pd.Timestamp(datetime.now(timezone.utc))`. Log dependency-call exceptions instead of silently returning a degraded default.
- **Schwab `price_history` defaults to `(periodType=day, period=10, frequencyType=minute, frequency=1)`** — 10 days of 1-MINUTE bars, not daily. Any consumer wanting DAILY bars MUST pass `(year|month, N, daily, 1)` explicitly. Discriminating test asserts the kwargs via mock/signature.
- **OHLCV fetch scope = open-trade tickers ONLY.** `build_dashboard` computes `ohlcv_tickers = sorted({t.ticker for t in open_trades})` — do NOT union watchlist/active_tickers (burns quota, trips the breaker). Skip the fetch entirely with zero open trades.
- **External-API empty result must be treated as transient in write-through caches/archives** (F6). yfinance/broker/weather APIs return empty for rate-limit/network/weekend reasons — NEVER overwrite cached content with an empty response: retain existing, leave stale metadata so the next call retries, never blank. Applies to `swing/data/ohlcv_archive.py` + any future write-through cache. When the cache helper accepts a **dataclass**, enforce non-empty at the `__post_init__` construction barrier (rejecting in the helper is too late — the row is already built).
- **Shared-infrastructure cache hooks must return the FULL archive; consumers slice.** A fallback that pre-truncates to one consumer's minimum window silently caps every other consumer (e.g. chart needs ~145 business days vs SMA50's 60). Return full history; let each consumer slice.
- **(F6-addendum, Arc 8) Trailing-bar NaN-Close: trim, don't append.** yfinance can return the NEWEST bar with `Close=NaN` while O/H/L/V are present (the adjusted-close derivation artifact — DISTINCT from the all-NaN F6 case). The gap fetch starts at `latest_stored+1` so a stored ragged bar never self-heals until the ≤13-day staggered full refresh → NaN poisons SMAs/weather/charts for days (run #99, 2026-06-10: 134 archives, manual repair). **Fix:** the shared `_trim_trailing_ragged(df)` drops trailing rows where ANY of OHLC is NaN (Volume-only-NaN exempt; INTERIOR ragged rows PRESERVED — the Phase-15 bad-bar-accept posture for HISTORICAL bars is unchanged), applied at TWO sites — the serial `_yf_download_window` return (covers both serial branches) + the warm `_warm_one_window` per-ticker loop (covers both warm cohorts; trim-to-empty skips without fallback). Trim-to-empty composes with the existing empty-fetch guards (no write, meta stale, retry next call). Audit: per-trim `log.warning` (→ `pipeline.log`) + `WarmReport.trailing_nan_trimmed` → the runner #27 warning entry. Build tests from the REAL #99 shape (NaN Close, O/H/L/V present) — the all-NaN F6 case must still pass separately; the interior-NaN-preserved discriminator FAILS an over-eager drop-all-NaN-rows impl.

### Web / HTMX / templates / forms

- **HTMX 4xx fragments need an explicit config override.** HTMX 2.x default (`{code:"[45]..",swap:false}`) fires the error event but doesn't swap. `base.html.j2` has an `htmx.config.responseHandling` override enabling 4xx swapping — preserve it. TestClient won't catch a regression (it asserts body, not DOM).
- **Starlette middleware is LIFO.** `add_middleware` order is the REVERSE of request execution — later calls become more-outer. Security-critical middleware (OriginGuard strict-mode, request-id) must be added LAST so it wraps everything.
- **`base.html.j2` is shared — a new `vm.foo` field requires adding it (with a safe default) to EVERY base-layout VM** (`DashboardVM`, `PipelineVM`, `JournalVM`, `WatchlistVM`, `PageErrorVM`) or Jinja 500s unrelated routes with `UndefinedError`.
- **Cache + executor race: workers must not write shared state when the request thread cancels on deadline miss.** `Future.cancel()` is a no-op on a running worker. Worker returns a pure value; the request thread writes to `_store` ONLY for futures done in-deadline (`wait(timeout=deadline)`, iterate `done` not `pending`).
- **HTMX OOB-swap partials that hand-duplicate full-page markup drift silently.** OOB partials and full-page render must go through the SAME `{% include %}` target, never hand-duplicated fragments.

### SQLite / transactions / migrations / schema

- **`os.replace` requires same filesystem.** On Windows with Drive-synced dest + `$TMP` on another volume it raises `OSError 18`. Create temp files in the DESTINATION dir (`NamedTemporaryFile(dir=<dest_dir>)`). Never `shutil.move` expecting overwrite on Windows.
- **Python `... or ""` collides with SQL CHECK-constraint nullability** when the enum excludes empty string. Use `... or None` for nullable text columns with CHECK constraints; add a test submitting an empty field asserting NULL persisted.
- **SQLite `INSERT OR REPLACE` / bare `REPLACE` is `DELETE old + INSERT new`** — cascade-wipes child rows via `ON DELETE CASCADE`, issues a NEW PK, rewrites history. For audit/uniqueness tables use SELECT-then-UPDATE-or-INSERT.
- **Service-layer `with conn:` opens its own transaction** — do NOT call it from inside an outer single-transaction (inner context-manager commits the connection's single transaction). When composing multi-step atomic side-effects, call the REPO-level functions directly; preserve the repo-vs-service asymmetry.

- **Reconciliation flow pivot uses SAVEPOINT-per-discrepancy inside the outer run transaction** (`SAVEPOINT correction_sp_<id>` / RELEASE on success / ROLLBACK TO+RELEASE on exception) — without it a partial mutation commits on the outer COMMIT.
- **SELECT-first idempotency MUST precede payload validation** — a terminal-state row must return its existing audit-row id even with a stale/None payload. Step order: sandbox short-circuit → SELECT + None-check → terminal-state idempotent return → payload validation → atomic write.

- **Queries `ORDER BY started_ts DESC` on `pipeline_runs` mask prior completes mid-run** (a NULL-`finished_ts` row wins). Two-read pattern: most-recent COMPLETED row's `finished_ts` for "last good data"; most-recent-started row's `state` for "what's happening now".
