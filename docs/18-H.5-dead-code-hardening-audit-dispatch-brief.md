# 18-H.5 — "Hardening into dead code" audit (the treadmill's retroactive twin)

**Authored:** 2026-06-15 by CHARC (operator-directed, a consequence of the 18-D defensive-hardening treadmill). **Container:** Phase 18 §18-H catch-all (18-H.5). **HOLD dispatch until -H is commissioned** (operator standing direction). **Cycle: AUDIT-FIRST** — Phase 1 read-only catalog → Phase 2 scoped remediation (separate dispatch on the operator-approved subset). **Cell rec:** Phase 1 = a read-only audit pass (Explore/audit agent or `implementer-opus-high`); Phase 2 = `implementer-opus-high` (judgment-dense — the cite-the-constraint rigor). **Review tier:** Phase 2 executing = `review-strong`.

## 1. Mandate
The treadmill corollary (recipe §3, `94ac2905`) PREVENTS *future* hardening into dead code. This audit finds + removes what is ALREADY in the codebase: **defensive branches that guard a state an upstream constraint VERIFIABLY prevents** (a schema `CHECK`/`UNIQUE`/`NOT NULL`/FK, a migration, or a constrained writer/repo), so the branch is UNREACHABLE — its test (if any) just *skips* on the constraint. Dead-code hardening is a maintenance tax + a cascade-regression surface (the R11 lesson). Remove it — but SAFELY (§4).

## 2. Phase 1 — the read-only AUDIT (primary deliverable; low risk)
Enumerate candidate dead-code-hardening cases. **Scope: READERS / CONSUMERS / MONITORS** (the pattern's home — NOT writers, which DO enforce the schema): `swing/monitoring/` (`tool_health`, `research_health` once it lands, `stoplights`), `swing/metrics/`, `swing/web/view_models/` + `routes/` read paths, `swing/journal/` (`analyze`/`stats`), the pipeline READ paths. NOT a blind sweep of every `try/except` (many defenses are legitimate) — target the schema-prevented-value pattern.

For EACH candidate, a catalog entry: (a) the file:line + the defensive branch; (b) **the CITED upstream constraint** — the migration/`CHECK`/`UNIQUE`/`NOT NULL`/FK line, **READ to verify it actually prevents the value** (the 18-E never-assume rule); (c) the classification:
- **(A) redundant** — a GENERAL handler already covers it → the per-case branch is removable.
- **(B) per-case → general** — N per-case branches that should be ONE general graceful-degrade + LOGGING.
- **(C) NOT dead — KEEP** — the constraint is weaker than assumed, OR a non-schema write path exists, OR it guards a genuinely-unconstrained input (missing/pre-schema table, absent/corrupt FILE, future-dated/out-of-contract artifact, non-schema-prevented reachable vector). **The DEFAULT is C** — a candidate is A/B ONLY with a verified constraint citation.

Output: the catalog (a doc or a gitignored findings file). **CHARC + operator review the catalog and decide which A/B candidates to remediate** (the Phase-2 scope). No code shipped in Phase 1.

## 3. Phase 2 — scoped remediation (separate dispatch, on the approved catalog subset)
For the operator-approved A/B candidates, per the corollary: **replace per-case dead branches with a GENERAL graceful-degrade + LOGGING** (preserve the safety net — do NOT merely delete defenses), or remove a branch redundant with an existing general handler. TDD: remove the skip-on-constraint test; add a general-handler test (a genuinely-REACHABLE error → graceful degrade + logged). The discriminating test proves the removed branch was DEAD (the cited constraint makes its state unreachable) AND the general handler covers the reachable cases.

## 4. LOCKS / risks (the load-bearing part)
- **Citation rigor:** a candidate is A/B ONLY with a cited + READ constraint that verifiably prevents the value. Absent/unverified → C (KEEP). NEVER remove a non-verifiably-dead defense — that converts a real hole into a removed guard (the inverse failure of the treadmill, and worse).
- **Preserve the safety net:** the corollary's positive half — a reader STILL needs a general graceful-degrade + logging to identify a hole if one ever opens (a future migration relaxing a constraint; a manual DB edit). Replace per-case WITH general; don't strip the defense to nothing.
- **Still-guard the unconstrained:** class C (missing/pre-schema tables, files, future-dated artifacts, non-schema-prevented vectors) STAYS untouched — this audit targets ONLY schema-verifiably-prevented guards.
- **Phase-isolation:** Phase 1 is READ-ONLY (no tripwire). Phase 2's tripwire is CATALOG-DEPENDENT — if remediation touches `swing/data` or `swing/trades` (a reader there), it routes through CHARC (the read-only-lock; removing dead code is a cleanup, not a schema/behavior carve-out, but the locked trees want a CHARC pass). The Phase-2 dispatch states its file list + the tripwire disposition.

## 5. Gate
- **Phase 1 (audit):** CHARC + operator review the catalog (read-only — no code gate).
- **Phase 2 (remediation):** orchestrator QA + (if `swing/data`/`trades` touched) a CHARC pass + **the no-false-green merged-head suite is the binding proof the removed branches were truly dead** (removing them breaks NOTHING) + ruff. `review-strong`.

## 6. Return report
Phase 1: the catalog (candidates + cited constraints + A/B/C classifications + counts; the surfaces swept + any deliberately-out-of-scope). Phase 2: the remediated cases (file:line), the general-handler replacements, the removed/added tests, the suite-green proof, Codex rounds + verdict.
